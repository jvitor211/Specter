# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability in Specter, **do not open a public issue**.

Instead, please report it responsibly:

1. **Email**: security@specter.dev
2. **Subject**: `[VULN] Brief description`
3. **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Impact assessment
   - Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 5 business days
- **Fix timeline**: Depends on severity
  - Critical: 24-48 hours
  - High: 7 days
  - Medium: 30 days
  - Low: Next release

## Scope

The following are in scope:
- Specter API (`specter/api/`)
- Authentication and API key management
- Package scanning and scoring logic
- VS Code extension
- Dashboard application

Out of scope:
- Third-party dependencies (report upstream)
- Social engineering attacks
- DDoS attacks
